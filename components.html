Alerts

Provide contextual feedback messages for typical user actions with the handful of available and flexible alert messages.

Alerts comes with two(2) kinds, simple alerts and alerts with action buttons

Alert Class
Simple Alert .alert alert-simple
Primary Alert .alert alert-primary
Success Alert .alert alert-success
Warning Alert .alert alert-secondary
Error Alert .alert alert-error
Simple Alerts

Simple Alerts is any feedback message container that does not require any feedback or response from users. Use the proper styling class alert alert-simple.

This is a simple feedback message that does not require user feedback or response.

                        				
                              
<div class="alert alert-simple d-flex align-items-start" role="alert" >
    <div class="">
       <p class="alert-text" >
        This is a simple feedback message that does not require user feedback or response.
        </p >
    </div>
</div>
Alerts with Action Buttons

Alert messages might comes with action buttons for the users feedback or response. For proper styling, use any of the contextual classes applicable to the message.

Primary Alert
Information Message

Conveying an important message should be noticebly visible and yet not disruptive to core contents. It is a good practice to always offer suggestions for user actions, you may use control input button and other form controls. 

																 
<div class="alert alert-primary  d-flex align-items-start" role="alert">
  <div class="alert-icon" >
    <i class="bi bi-info-circle-fill" > </i>
  </div >
    <div class="alert-message">
        <h5 class="alert-title-primary" > Information Message </h5>
        <p class="alert-text" >
          Conveying an important message should be noticebly visible and yet not disruptive to core contents.
          It is a good practice to always offer suggestions for user actions, you may use control input button and other form controls.
        </p >
        <button type="button" class="btn btn-sm btn-primary my-3" >Close </button>
    </div >
</div >
Variations

The following are variation of alert messages styles available for use.

Success Alert

To provide feedback for any successful data updates, use the class .alert alert-success to wrap your message if you intend to warn users.

Success Message

Providing feedback after a data changes or system tasks is essential for users. 

																	
<div class="alert alert-success d-flex align-items-start" role="alert" >
  <div class="alert-icon" >
    <i class="bi bi-check-circle-fill" > </i >
  </div >
  <div class="alert-message" >
     <h5 class="alert-title-success" >Success Message </h5 >
     <p class="alert-text" >
        Providing feedback after a data changes or system tasks is essential for users.
     </p >
     <button type="button" class="btn btn-sm btn-success my-3" >OK </button >
    </div >
</div >
Warning Alert

Warning alerts are properly used to warned and requires a response from the users. Use the class .alert alert-secondary to wrap your message if you intend to warn users.

Warning Message

Confirming user response is essential for data updates. 

																	
<div class="alert alert-secondary d-flex align-items-start" role="alert">
  <div class="alert-icon" >
      <i class="bi bi-exclamation-triangle" > </i>
  </div >
  <div class="alert-message">
    <h5 class="alert-title-secondary" >Warning Message </h5>
    <p class="alert-text">
        Confirming user response is essential for data updates.
    </p>
    <button type="button" class="btn btn-sm btn-primary my-3" >Yes </button>
    <button type="button" class="btn btn-sm btn-primary-outline my-3" >Cancel </button>
  </div>
</div>
Error Alert

Error alerts are recommended to be used for critical errors for users. Use the class .alert alert-error to wrap your message if you intend to inform users for errors related.

Error Message

Informing system error is important to provide feedback for a specific task such as data updates. 

																	
<div class="alert alert-error d-flex align-items-start" role="alert">

    <i class="bi bi-x-octagon"></i >
  </div >
  <div class="alert-message-error" >
    <h5 class="alert-title-error" > Error Message </h5 >
    <p class="alert-text" >
      Informing system error is important to provide feedback for a specific task such as data updates.
    </p >
    <button type="button" class="btn btn-sm btn-error my-3" >OK </button >
    </div >
    </div >

Outline Buttons

The outline buttons is an alternative variants for the filled button.Replace the default modifier classes with the .btn-*-outline ones to remove all background images and colors on any button. Applicable for colored background for contrast and provides a context for the user. 

                      
  <button type="button" class="btn btn-primary-outline m-1">Primary</button>
  <button type="button" class="btn btn-warning-outline m-1">Warning</button>
  <button type="button" class="btn btn-error-outline m-1">Error</button>
  <button type="button" class="btn btn-success-outline m-1">Success</button>
  <button type="button" class="btn btn-dark-outline m-1">Dark</button>
Icon Buttons

Icon buttons is best used for smaller screens such as mobile or tablet devices where space is limited. Refer to the Icons library for available icons. To use an icon buttons, wrapped the button with an avalibale icon class <i> class="bi bi-star"></i>. 

                      
  <button type="button" class="btn btn-primary m-1"><i class="bi bi-star"></i></button>
  <button type="button" class="btn btn-warning m-1"><i class="bi bi-exclamation-triangle"></i></button>
  <button type="button" class="btn btn-error m-1"><i class="bi bi-trash3"></i></button>
  <button type="button" class="btn btn-success m-1"><i class="bi bi-check-circle"></i></button>
  <button type="button" class="btn btn-dark m-1"><i class="bi bi-download"></i></button>
  <button type="button" class="btn btn-light m-1"><i class="bi bi-arrow-right"></i></button>

*Outline buttons can also be used as long as the background of the buttons are non-white to provide clear context of the buttons.

Button Sizes

For larger or smaller buttons? Add .btn-lg or .btn-sm 

                      
        <button type="button" class="btn btn-lg btn-primary m-1">Large Primary Button</button>
        <button type="button" class="btn btn-lg btn-warning m-1">Large Warning Button</button>
        
                    
                      
        <button type="button" class="btn btn-lg btn-primary m-1">Large Primary Button</button>
        <button type="button" class="btn btn-lg btn-warning m-1">Large Dark Button</button>
Disabled Button

To add styles for disabled button, add disabled. 

                      
  <button type="button" class="btn btn-primary m-1 disabled"><i class="bi bi-star"></i></button>
  <button type="button" class="btn btn-primary m-1 disabled">Primary</button>

Modals

Modals are used to bring the attention of a site visitor to an important thing he/she needs to take action on. And unless the action is taken or the modal window is closed, the visitor cannot continue interacting with the page.

Modals are of diffent types and sizes

Modal Use
Alert Modal Used to call the user's attention to something important
Confirmation Modal Used to confirm user's choice for an action
Scrollable Modal It is use to provide an important information where the content is longer that the viewport of the device
Alert Modal

An Alert or simple information modal is used to call the attention of a user to an important thing.

Preview
Modal title
Note that all activities on this site are being tracked
Code
																	

<div class="modal fade" id="exampleModal" tabindex="-1" aria-labelledby="exampleModalLabel" aria-hidden="true">
  <div class="modal-dialog">
    <div class="modal-content">
      <div class="modal-header">
      <h5 class="modal-title" id="exampleModalLabel">Modal title</h5>
      <button type="button" class="btn-close" data-bs-dismiss="modal" aria-label="Close"></button>
      </div>
      <div class="modal-body">
        Note that all activities on this site are being tracked
      </div>
      <div class="modal-footer">
        <button type="button" class="btn btn-secondary" data-bs-dismiss="modal">Close</button>
        <button type="button" class="btn btn-primary">Save changes</button>
      </div>
    </div>
  </div>
</div>

Spinners

A way to provide feedback to user that a certain action is currently being done by the system.

Variations
Border Spinner
Button Spinners
Spinner Sizes
Border Spinner

The border spinner is used when the section page is enough and not limited. Consider the appropriate sizes of the containers or screens when using a spinner.

                            
  <div class="spinner-primary" role="status">
    <span class="sr-only">Loading...</span>
  </div>
Button Spinners

Use spinners within buttons to indicate an action is currently processing or taking place. You may also swap the text out of the spinner element and utilize button text as needed. 

                      
  <button class="btn btn-primary" type="button" disabled>
    <span class="spinner-border spinner-border-sm" role="status" aria-hidden="true"></span>
      <span class="sr-only"</span>
  </div>

  <button class="btn btn-primary" type="button" disabled>
    <span class="spinner-border spinner-border-sm" role="status" aria-hidden="true">Loading...</span>
  </button>
Spinner Sizes

Customize spinner sizes by adding .spinner-sm or .spinner-lg to alter the spinner dimension as needed. Smaller spinner is mainly used for charts with small window of display whereas larger spinner for the whole page loading.
